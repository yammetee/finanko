# MVP TODO: бухгалтерия и математика Finanko

Этот документ заменяет прежний широкий план. MVP Finanko не должен включать лишние учетные сценарии.

## MVP scope

Нужно поддержать только базовое личное ведение бюджета:

- счета и портфели;
- доходы, например зарплата;
- ежедневные расходы;
- категории расходов и доходов;
- чеки с позициями;
- регулярные доходы/расходы;
- кредиты, долги, ипотека;
- платеж по долгу с разделением на тело долга и проценты;
- явные начисления процентов;
- отображение сумм в выбранной валюте через конвертацию для отчетов.

Не нужно для MVP:

- transfers между счетами;
- обмен валюты как операция;
- import/export backup;
- полноценная сложная double-entry бухгалтерия в UI;
- сложные инвестиционные сделки, лоты, PnL, комиссии брокера;
- бюджетирование по планам, лимитам и прогнозам.

## Что нужно доделать

- Добавлены тесты на несколько платежей по одному долгу.
- Проверен сценарий: платеж частично гасит тело долга, проценты идут в расходы.

- Проверено, что `debtTotal` считается как абсолютная сумма liability-счетов.

### 2. Проценты

- Зафиксирована day count policy для MVP: `365`.
- Зафиксирована rounding policy: округлять каждое начисление до minor units.
- Проверены daily/monthly начисления на отрицательных и положительных счетах.
- Добавлена защита от начисления процентов на нулевой баланс.

### 3. Валюты только для отображения

- Конвертация валют используется только для отображения и аналитики.
- Операции обмена валюты не добавлены.
- Исторические отчеты используют курс на дату операции или последний известный курс до даты.
- Добавлены тесты на display-only конвертацию, live-rate same-day, historical fallback и cross-currency display.

### 4. Data layer

- Источником истины является реляционная схема Supabase с constraints, indexes и RLS.
- Локальное хранилище финансовых данных не используется.
- Изменения сначала подтверждаются базой и только после этого отражаются в UI state.

### 5. Удаление устаревшей логики

- Удалены неиспользуемые repository-заготовки локального слоя.
- Удалены не-MVP ledger kinds/roles: fee, refund, currency revaluation, rounding.
- Удален неиспользуемый `savingsTotal` из analytics.
- Не держать рядом две финансовые модели, если одна признана неправильной.

## Definition of Done для MVP бухгалтерии

- `income` увеличивает доходы и активный счет.
- `expense` увеличивает расходы и уменьшает активный счет.
- `debt_payment` уменьшает активный счет, уменьшает долг на principal и относит interest в расходы.
- `interest_accrual` создает явную операцию.
- Чеки по позициям сходятся с общей суммой.
- Категории расходов по чекам считаются по позициям.
- Валюты конвертируются только для отображения.
- Нет transfers.
- Нет import/export backup в MVP runtime.
- Нет старой неправильной процентной логики.
- `npm test`, `npm run build`, `npm run lint` проходят.
